# 多账号系统使用说明

## 🚀 快速开始

### 1. 环境准备
```bash
# 安装依赖
pip install -r requirements.txt

# 确保有 .env 配置文件
# 如果没有，请复制并编辑 .env.example
```

### 2. 配置迁移（首次使用）
如果你之前使用过单账号版本：
```bash
# 运行配置迁移工具
python migrate_config.py
```

### 3. 启动系统
```bash
# 启动多账号系统
python main_multi.py
```

### 4. Web管理界面
访问 http://localhost:5000 进入管理界面

## 📋 功能说明

### 账号管理
- **添加账号**：在Web界面中添加多个闲鱼账号
- **配置管理**：每个账号独立配置提示词和功能开关
- **状态监控**：实时查看各账号运行状态
- **批量操作**：支持批量启动/停止账号

### 提示词配置
每个账号可以独立配置5类提示词：
- **意图分类**：识别用户消息类型
- **价格谈判**：处理议价相关消息
- **技术咨询**：回答产品技术问题
- **默认回复**：处理一般性对话
- **买家回复**：作为买家时的对话策略

### 功能开关
- **卖家回复**：启用后作为卖家自动回复买家
- **买家回复**：启用后作为买家自动与卖家沟通

## 🎯 使用流程

### 1. 添加第一个账号
1. 点击"添加账号"
2. 填写账号名称、用户ID和Cookie
3. 选择启用的功能（卖家/买家回复）
4. 点击"创建账号"

### 2. 配置提示词
1. 进入账号配置页面
2. 在"提示词配置"标签页中编辑各类提示词
3. 点击"保存所有提示词"

### 3. 启动账号
1. 在账号列表中点击"启动"按钮
2. 或使用"启动所有账号"批量启动
3. 在监控页面查看运行状态

## 🔧 高级配置

### 环境变量配置
在 `.env` 文件中可配置：
```bash
# Web界面配置
WEB_HOST=0.0.0.0
WEB_PORT=5000
WEB_DEBUG=False

# AI模型配置
API_KEY=your_api_key
MODEL_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MODEL_NAME=qwen-max

# 系统参数
HEARTBEAT_INTERVAL=15
TOKEN_REFRESH_INTERVAL=3600
MANUAL_MODE_TIMEOUT=3600
LOG_LEVEL=INFO

# 扣子智能体配置（可选）
COZE_API_KEY=your_coze_api_key
COZE_BUYER_BOT_ID=your_buyer_bot_id
```

### 数据库配置
- 每个账号使用独立的数据库文件
- 位置：`data/chat_history_{account_id}.db`
- 支持自动备份和恢复

### 人工接管模式
- 在聊天中发送句号（.）可切换人工接管模式
- 人工接管期间AI不会自动回复
- 超时后自动恢复AI回复

## 📊 监控和日志

### 实时监控
- 访问监控页面查看所有账号状态
- 支持自动刷新和实时日志
- 显示连接状态、错误次数等信息

### 日志管理
- 日志级别：DEBUG、INFO、WARNING、ERROR
- 支持实时查看和历史记录
- 按账号分别记录运行情况

## 🔒 安全注意事项

1. **Cookie安全**：请勿将Cookie信息泄露给他人
2. **访问控制**：Web界面默认监听所有IP，生产环境请修改WEB_HOST
3. **数据备份**：定期备份数据库文件和配置
4. **账号限制**：避免同时运行过多账号，防止被平台限制

## 🐛 常见问题

### Q: 账号连接失败怎么办？
A: 检查Cookie是否过期，重新获取并更新

### Q: AI回复不符合期望？
A: 在账号配置页面中调整对应的提示词

### Q: 如何备份数据？
A: 复制`data`目录下的所有.db文件

### Q: Web界面无法访问？
A: 检查端口5000是否被占用，或修改WEB_PORT环境变量

### Q: 内存占用过高？
A: 减少同时运行的账号数量，或调整日志级别为WARNING
