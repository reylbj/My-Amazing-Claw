## 六、使用场景示例

### 场景1：公众号内容生产

**流程**：
1. 用户发送 `今日选题`
2. 选题官调用Pro，1次请求输出15个选题
3. 用户回复编号（如 `3`）
4. 选题官调用Pro，输出1000-3000字初稿
5. 自动推送到微信公众号草稿箱

**时间节省**：从2小时压缩到1分钟
**配额消耗**：2次Pro/天

### 场景2：AI行业情报

**流程**：
1. 每日08:00自动触发
2. AI战略官抓取Twitter/36氪/HuggingFace/The Verge
3. 筛选Top10条，调用Flash生成结构化简报
4. 推送到用户指定渠道

**时间节省**：从1小时压缩到自动化
**配额消耗**：1次Flash/天

### 场景3：短视频批量生产

**流程**：
1. 用户发送 `写脚本 + 主题`
2. 短视频编导调用Pro，1次输出完整脚本（标题+钩子+分镜+口播+CTA）
3. 用户可直接拍摄

**时间节省**：从30分钟压缩到1分钟
**配额消耗**：1次Pro/条

## 八、已知问题与解决方案

### 问题：桌面重复出现"小龙虾🦞"文件夹

**现象**：每次运行项目时，桌面会多出一个名为"小龙虾🦞"的文件夹

**根本原因**：
OpenClaw主配置文件 `~/.openclaw/openclaw.json` 中的 `workspace` 路径指向了旧的 `小龙虾🦞` 文件夹（项目初始化时自动创建），而用户实际使用的是 `家养小龙虾🦞` 文件夹。

**解决方案**：
1. ✅ 修改 `~/.openclaw/openclaw.json` 中的 workspace 路径
   - 旧路径：`/Users/a8/Desktop/小龙虾🦞/openclaw-workspace`
   - 新路径：`/Users/a8/Desktop/家养小龙虾🦞/openclaw-workspace`

2. ✅ 将旧文件夹重命名为备份
   ```bash
   mv ~/Desktop/小龙虾🦞 ~/Desktop/小龙虾🦞_old_backup_可删除
   ```

3. ✅ 验证配置生效
   ```bash
   python3 -c "import json; print(json.load(open('~/.openclaw/openclaw.json'))['agents']['defaults']['workspace'])"
   ```

**状态**：✅ 已解决，下次运行项目不会再创建重复文件夹
