## 二、技术突破

### 1. 微信API中文编码问题
**问题**：草稿箱显示Unicode转义码（`\u5c0f\u9f99\u867e`）
**原因**：`requests.post(json=data)` 默认 `ensure_ascii=True`
**解决**：手动序列化JSON + UTF-8编码
```python
data=json.dumps(payload, ensure_ascii=False).encode('utf-8'),
headers={'Content-Type': 'application/json; charset=utf-8'}
```

### 2. 微信草稿API限制
- **标题**：最大64字节（约21个中文字符）
- **摘要**：最大54字节（约18个中文字符）
- **封面图**：必填，需先上传获取 `thumb_media_id`
- **解决**：使用公众号已有素材库中的图片

### 3. 桌面文件夹重复问题
**根本原因**：两处配置指向旧路径
- `~/.openclaw/openclaw.json` → `agents.defaults.workspace`
- `~/.openclaw/agents/main/sessions/sessions.json` → `workspaceDir` + 15个文件路径

**解决**：Python脚本递归替换所有路径引用

## 七、已知限制与解决方案

### 1. WhatsApp触发词识别问题（已解决）

**问题**：发送"今日选题"后，AI回复了但没有返回15个选题
**根本原因**：
1. OpenClaw读取的是根目录 `AGENTS.md`，而不是子文件夹中的配置文件
2. 主力模型配置在 `~/.openclaw/openclaw.json` 而非 `config/api_config.yaml`
3. 旧会话持久化导致新配置不生效

**完整解决步骤**：
1. 在根目录 `AGENTS.md` 顶部添加触发词识别逻辑
2. 修改 `~/.openclaw/openclaw.json` 中 `primary` 字段为 Pro 模型
3. 删除旧的 WhatsApp session（`~/.openclaw/agents/main/sessions/sessions.json` 中移除对应key）
4. 重启 gateway：`openclaw gateway restart`

**验证**：日志显示 `agent model: google/gemini-2.5-pro-preview-05-06` ✅

**关键发现**：
- OpenClaw注入的workspace文件：`AGENTS.md`, `SOUL.md`, `USER.md`, `TOOLS.md`, `IDENTITY.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`
- 这些文件必须在workspace根目录，不能在子文件夹
- AI员工配置需要写在根目录的 `AGENTS.md` 中才能被识别
- **模型配置优先级**：`openclaw.json` > `api_config.yaml`（openclaw.json优先级更高）
