# 🐟多账号自动回复系统 - 管理脚本使用说明

## 📦 脚本文件
- `start.sh` - 启动系统
- `stop.sh` - 停止系统  
- `status.sh` - 查看系统状态

## 🚀 使用方法

### 启动系统
```bash
./start.sh
```
**功能**:
- 自动检查端口占用并清理
- 后台启动系统
- 创建PID文件用于管理
- 输出系统状态和访问地址

### 停止系统  
```bash
./stop.sh
```
**功能**:
- 优雅关闭系统(SIGTERM)
- 10秒超时后强制关闭(SIGKILL)
- 清理PID文件和端口占用
- 确保完全停止

### 查看状态
```bash
./status.sh
```
**功能**:
- 显示进程运行状态
- 检查端口占用情况  
- 测试Web服务可用性
- 显示最近日志
- 提供管理命令提示

### 查看实时日志
```bash
tail -f logs/system.log
```

## 📋 系统信息
- **Web界面**: http://localhost:5002
- **日志文件**: logs/system.log  
- **PID文件**: xianyu_multi.pid
- **端口**: 5002

## ⚠️ 注意事项
1. 首次运行前确保配置文件 `.env` 已正确设置
2. 如果系统异常退出，运行 `./stop.sh` 清理残留进程
3. 端口5002被占用时会自动清理，但可能需要管理员权限
4. 日志文件会持续增长，建议定期清理

## 🔧 故障排除
1. **启动失败**: 检查 `.env` 文件和必要的依赖
2. **端口占用**: 运行 `./stop.sh` 清理或重启系统
3. **Web界面无法访问**: 运行 `./status.sh` 检查服务状态  
4. **系统无响应**: 运行 `./stop.sh` 强制停止后重新启动
