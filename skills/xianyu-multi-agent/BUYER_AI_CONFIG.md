# 🛒 买家AI系统配置指南 (V3.0 - 扣子智能体版)

## 🆕 V3.0 新特性

✅ **扣子智能体**: 买家AI升级为扣子智能体，对话更自然  
✅ **双引擎架构**: 买家用扣子，卖家用通义千问，各司其职  
✅ **智能Fallback**: 扣子失败时自动切换到通义千问  
✅ **无缝集成**: 原有功能完全保持，只是买家更聪明了
✅ **🔧会话连续性**: 修复了会话管理问题，同一商品的对话现在能保持连续性

## 🤖 扣子智能体配置

### 快速配置

在你的 `.env` 文件中添加：

```bash
# 扣子API配置（买家AI专用）
COZE_API_KEY=your_coze_api_key_here          # 扣子API密钥
COZE_BASE_URL=https://api.coze.cn            # 扣子API地址  
COZE_BUYER_BOT_ID=your_buyer_bot_id_here     # 买家智能体ID

# 原有配置保持不变（卖家AI继续使用）
API_KEY=your_openai_api_key                   
MODEL_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
MODEL_NAME=qwen-max

# 买家AI系统配置
BUYER_MAX_BUDGET=5000          # 买家最大预算(元)
BUYER_TARGET_DISCOUNT=0.8      # 目标折扣比例(0.8=8折)
BUYER_MAX_ROUNDS=3             # 最大议价轮次
BUYER_PATIENCE=2               # 耐心等级(1-3，影响等待时间)
```

## 🔄 会话分离机制

### 问题分析
**你的问题很对！** 之前确实有会话管理缺陷：
- ❌ 每次调用扣子都是新会话
- ❌ 智能体无法记住之前的对话
- ❌ 重复询问相同问题

### 解决方案  
现在采用**智能会话ID生成**：

```python
conversation_id = f"buyer_{item_id}_{hash(chat_id) % 1000}"
```

### 会话分离逻辑

| 分离级别 | 规则 | 效果 |
|---------|------|------|
| **商品级别** | 不同商品ID → 不同扣子会话 | 商品A和商品B的对话完全分开 |
| **用户级别** | 不同聊天ID → 不同扣子会话 | 张三和李四与同一商品的对话分开 |
| **角色级别** | 买家用扣子，卖家用通义千问 | 双引擎架构，各司其职 |

### 实际效果对比

**🔧 修复前（V2.0）**:

买家: "嗯，东西成色怎么样？"

第2轮 - conversation_id: buyer_hash2  (新会话!)

```

**✅ 修复后（V3.0）**:
```

## 买家AI工作流程

1. **角色识别**: 自动判断是否为程序自己的商品
   - 如果是自己的商品 → 卖家模式（原有功能）
   - 如果不是自己的商品 → 买家模式（新功能）

2. **买家AI阶段**:
   - **问询阶段**: 询问商品详情、成色、发货等
   - **议价阶段**: 根据配置进行价格谈判
   - **决策阶段**: 最终决定是否购买

3. **防重复机制**: 避免短时间内发送相同类型消息

## 使用说明

买家AI系统会自动在以下情况下激活：
- 收到不是自己商品的消息时
- 程序作为买家与卖家对话

系统特点：
- 智能识别卖家回复类型
- 分阶段进行对话管理
- 基于配置的理性议价策略
