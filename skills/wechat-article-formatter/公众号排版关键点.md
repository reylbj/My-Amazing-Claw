# 公众号排版关键点速查

## 橙韵风格CSS兼容性铁律
1. **白色卡片结构**: 必须用`<table><tr><td>`替代`<section>`+`box-shadow`(公众号不支持box-shadow)
2. **白色卡片内容**: 仅放hook(2-3句话)不放正文避免重复
3. **章节标题**: 用橙色渐变方块+中文序号(一二三四五)不用H2标签
4. **案例展示**: 用渐变背景table卡片不用列表
5. **推送脚本**: `wechat_draft.py`已修复HTML自动检测(检测`<section>`/`<p style=`标签)HTML直接推送不二次转换

## 验证清单
- ✅ 头部渐变区(橙色135度+胶囊标签+标题+日期)
- ✅ 白色卡片(table结构无box-shadow+仅hook内容)
- ✅ 章节标题(橙色方块序号+中文一二三四五)
- ✅ 案例卡片(渐变背景table结构)
- ✅ 无重复内容
- ✅ CSS完全兼容(无box-shadow/flex/grid)

## 推送流程
```bash
# 1. 生成Markdown文章(888-1888字)
# 2. 转换为橙韵风格HTML(按上述铁律)
# 3. 推送草稿箱
python3 scripts/wechat_draft.py --file "path/to/content.txt" --title "标题" --digest "摘要"
```

## 常见问题
**Q: 手动复制HTML正常但脚本推送格式错乱?**
A: 已修复。脚本现在自动检测HTML文件直接推送不转换。

**Q: 白色卡片显示异常?**
A: 检查是否用了box-shadow,必须改用table+td结构。

**Q: 章节标题格式不对?**
A: 必须用橙色渐变方块+中文序号,不能用H2标签。
