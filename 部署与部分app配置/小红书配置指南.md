# xiaohongshu-send 配置与验证指南

本指南把小红书发布流程固化为可执行脚本，目标是稳定完成「文本 + 图片」发布，并且在发布前后可观测（登录状态、MCP 连通性、发布结果）。

## 目录结构

```text
openclaw-workspace/
├── scripts/
│   ├── xiaohongshu_send_setup.sh   # 安装/登录/启动/状态/停止
│   └── xiaohongshu_send.py         # 参数校验/发布/端到端验证
└── xiaohongshu-send/
    ├── bin/                        # xiaohongshu-mcp / xiaohongshu-login
    ├── data/                       # cookies.json
    ├── logs/                       # mcp.log
    ├── run/                        # mcp.pid
    └── samples/
        └── publish.sample.json
```

## 端到端流程（对应 Step 0-6）

### Step 0: 准备环境

```bash
cd /Users/a8/Desktop/家养小龙虾🦞/openclaw-workspace
bash scripts/xiaohongshu_send_setup.sh setup
```

说明：
- 脚本会自动识别平台并下载最新 release（当前机器为 `Darwin arm64`）。
- 会安装到 `xiaohongshu-send/bin/`，并生成 `xiaohongshu-send/release.txt`。

### Step 1: 登录（首次或 cookies 失效时）

```bash
bash scripts/xiaohongshu_send_setup.sh login
```

说明：
- 会拉起浏览器扫码登录，成功后保存 cookies 到 `xiaohongshu-send/data/cookies.json`。
- 默认固定浏览器 profile：`xiaohongshu-send/profile`（可通过 `XHS_ROD_OPTIONS` 覆盖）。

### Step 2: 启动 MCP 服务

```bash
bash scripts/xiaohongshu_send_setup.sh start --port 18060 --headless true
```

调试模式（可见浏览器）：

```bash
bash scripts/xiaohongshu_send_setup.sh start --port 18060 --headless false
```

说明：
- 脚本会自动将 `/tmp/cookies.json` 统一链接到工作区 cookies，避免旧路径优先级导致“已登录却失效”。
- 自动生成备份：`xiaohongshu-send/data/cookies.backup.json`。

### Step 3: 检查登录状态

```bash
python3 scripts/xiaohongshu_send.py check-login --base-url http://127.0.0.1:18060
```

或：

```bash
bash scripts/xiaohongshu_send_setup.sh status --port 18060
```

### Step 4: 组织发布参数并校验

1) 准备 JSON（可参考 `xiaohongshu-send/samples/publish.sample.json`）  
2) 运行校验：

```bash
python3 scripts/xiaohongshu_send.py validate \
  --payload xiaohongshu-send/samples/publish.sample.json
```

校验规则：
- 标题长度 <= 20（与官方实现一致）
- 正文长度 <= 1000
- 图片必须是 URL 或本地绝对路径
- 标签建议 <= 10（超过会被服务端截断）
- `visibility` 值合法

### Step 5: 调用发布

先 dry-run（不提交）：

```bash
python3 scripts/xiaohongshu_send.py publish \
  --payload xiaohongshu-send/samples/publish.sample.json \
  --base-url http://127.0.0.1:18060 \
  --dry-run
```

实际发布：

```bash
python3 scripts/xiaohongshu_send.py publish \
  --payload /ABSOLUTE/PATH/TO/payload.json \
  --base-url http://127.0.0.1:18060
```

### Step 6: 校验结果

执行端到端验证（环境 + 服务 + MCP + 登录状态 + 可选 payload 校验）：

```bash
python3 scripts/xiaohongshu_send.py verify \
  --base-url http://127.0.0.1:18060 \
  --payload xiaohongshu-send/samples/publish.sample.json
```

停止服务：

```bash
bash scripts/xiaohongshu_send_setup.sh stop
```

## Windows 对照命令

你给的流程中 Windows 版本命令仍然成立：

```bash
bin/xiaohongshu-login-windows-amd64.exe
bin/xiaohongshu-mcp-windows-amd64.exe
```

本仓库脚本会按当前系统自动选取对应平台二进制。

## 质量与安全

- 不在正文中写敏感信息或隐私。
- 图片中避免个人信息、二维码、可追踪标识。
- 用占位符代替密钥（如 `TOKEN_REDACTED`）。
